# AGENTS.md

Development-agent entrypoint for the `evals` repository. Read this before doing anything;
then read the active roadmap and engineering standards.

`evals` is a **local OSS model evaluation harness**. Automated runs use LM Studio
only — no direct paid API calls. Baselines are imported from subs (ChatGPT, Claude,
Grok, Gemini) or Azure/Vertex credits collected outside the harness. Primary judging
is user review against those baselines; optional Azure credit judge is off by default.

## Repository Surfaces

- `registry/` — LM Studio model inventory, load-profile presets, judge/baseline policy.
- `baselines/` — imported reference outputs from subs/credits (not API-generated by harness).
- `suites/promptfoo/` — matrix eval configs, providers, prompts, workflow test cases.
- `suites/deepeval/` — local-only agent workflow smoke tests.
- `scripts/` — registry export, matrix orchestration, baseline import/compare, user-judge queue.
- `langfuse/` — self-hosted Langfuse OSS Docker Compose.
- `results/` — gitignored JSONL run artifacts and review queues.
- `docs/` — roadmaps, research, operations, engineering standards.

## Source Of Truth

- Live `lms ls --json` and LM Studio server state own model inventory.
- `registry/models.json` is a normalized export, not a substitute for refreshing.
- **Runtime facts (required before tuning):** `pnpm capture:system` → `results/system-profile.json`
  and `registry/runtime-snapshot.json`. Global CLI: `lm-runtime-snapshot` (installed from
  `C:\Users\james\tools\lm-runtime-snapshot`). Read `nvidia-smi`, `lms ps`, `lms load --estimate-only`,
  and `results/matrix-summary.json` — do not guess VRAM, presets, or placement.
- Phase B orchestrator/subagents must cite `optimization-state.json` + `system-profile.json` +
  latest matrix JSONL when changing `load-profiles.json` or suite thresholds.
- Locked decisions in the feasibility report override stale notes.
- Use `rg` first for repository search.
- Verify drift-prone facts against official docs (LM Studio, Promptfoo, DeepEval, Langfuse).

## Work Boundaries

- Evaluate models and workflows only. Do not implement Harness, Hermes, Zavi, or Studio runtimes.
- Tool-call evals use sandboxed callbacks — no real shell/network side effects in suites.
- Preserve unrelated user changes.
- Do not use destructive git commands unless explicitly requested.
- Keep changes scoped to the requested workstream.

## Local Workflow

- Windows dev host: PowerShell or cmd. Use `rg` for search.
- Copy `.env.example` → `.env` (Langfuse keys; optional Azure judge — off by default).
- Collect baselines separately: `pnpm baseline:prompts` → import → `pnpm matrix:smoke`.
- Langfuse (optional): `pnpm langfuse:up` — deferred; JSONL is primary
- Start LM Studio server (or let matrix runner load models via `lms`).
- Baselines (Vertex credits): `pnpm gemini:models` then `pnpm baseline:collect`
- Smoke matrix: `pnpm matrix:smoke` or `pnpm optimize:smoke`
- Phase B optimize: orchestrator assesses → subagents run (`docs/operations/continuous-optimization.md`)
- Checkpoint: `results/optimization-state.json`
- For goal runs with subagents, follow `docs/engineering/agents/orchestration-reliability.md`.

## Secrets And Safety

- Never commit secrets.
- No direct paid API calls from the harness. Do not wire `api.openai.com` keys.
- Optional Azure/Vertex judge uses credit-funded endpoints only, gated by env flags.
- Langfuse and optional Azure keys live in `.env` only.
- LM Studio local endpoint needs no API key.
- Never write tokens into suites, fixtures, results committed to git, or docs examples.

## Verification Ladder

- System snapshot: `pnpm capture:system` (or `pnpm capture:system:quick` before matrix)
- Global tool: `lm-runtime-snapshot -o snapshot.json --models-file registry/models.json`
- Registry: `pnpm registry:export`
- Baseline prompts: `pnpm baseline:prompts`
- Matrix smoke: `pnpm matrix:smoke`
- Compare: `pnpm compare:baseline`
- User-judge queue: `pnpm judge:queue`
- DeepEval: `pnpm eval:deepeval`
- Docs-only: read back Markdown + `git diff --check`
